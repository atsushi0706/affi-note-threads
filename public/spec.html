affi-note-threads

セミナー誘導などの LP（ランディングページ）を1枚貼るだけで、それを“紹介する”note記事と Threads 投稿を自動生成する、アフィリエイト専用ツール。どの分野の案件でも、LP を貼り替えれば使えます。

GitHub（コード全部） Gemini APIキー（無料）

これは何をするもの？

ひとことで言うと

LP を「元ネタ」にして、読者に価値提供しながら自然にその LP へ誘導する文章を、毎回ちがう切り口で量産するツール。投稿の自動化はせず、文章づくりまでを助けます（できた文章はコピーして手動で投稿）。

  • 汎用 ── LP を貼り替えるだけで、副業・AI・健康・メンタル…どの分野でも対応
  • BYO-key ── 利用者が各自の Gemini 無料キーを入れる。キーもプロフィールも各自のブラウザにだけ保存（サーバー保存なし）
  • 立ち位置は“紹介者” ── LP を書いた本人にはならない。読者に役立つ話をして、別枠で「興味あればこちら」と案内する
  • 毎回ちがう ── 同じ LP でも、生成のたびに切り口をランダムに振り、内容の重複を防ぐ

使い方（5ステップ）

利用者が画面でやること

1
Gemini APIキーを入れる

Google AI Studio で無料取得。ブラウザにだけ保存される。

2
紹介する LP の全文を貼る

これが記事の元ネタ。貼り替えれば別案件にも使える。

3
（任意）自分の設定を入れる

「あなたは何者か」「普段の言葉サンプル」「LPのURL」。書くほど“その人らしく”なる。各自のブラウザに自動保存。

4
方向性を5案出す → 1つ選ぶ

被りを防ぐため、書く角度を先に決める。選ばず“おまかせ”も可。🎲で振り直し。

5
生成 → コピーして投稿

note記事／Threads 5本／ピン止め3連投から選ぶ。出た文章をコピーして手動投稿。

内部の処理の流れ

ボタンを押すと、AI が段階を追って考える（チェーン方式・一撃生成はしない）

①
LP分析

LP全文から「ターゲット・悩み・提供価値・ジャンル・誘導ゴール」を抽出（JSON）。

②
Web検索リサーチ（毎回・自動）

アメリカ(英語圏)の最新情報を検索し、具体的なツール名・今すぐ試せる使い方を集める。Gemini の Google検索グラウンディング機能を使用。

③
方向性を5案提案

LPに一部つながる、互いに被らない切り口を5つ。ユーザーが1つ選ぶ（or おまかせ）。

④
モード別に生成

note / Threads / ピン止め のいずれかを、①〜③＋書き手設定＋ランダムな切り口をすべて材料に生成。

※ ①②③ の結果はキャッシュされ、同じLPなら使い回してAPIコストを節約。LPを変えるとやり直し。

3つの生成モード

note はリンクOK・Threads はURL非表示、と媒体の性質に合わせる

note記事

ONE HACK（H→A→C→K）構成の1本

  • 冒頭「こんにちは。この記事では〜」
  • 【案内ボックス】= CTAコピー＋LPのURL（別軸のお知らせ）
  • 本稿 = 価値提供（売り込まない）
  • 【案内ボックス】末尾にもう一度
  • 2000字前後・タグ5個
Threads 5本

1セット＝2投稿（フック＋リプライ）

  • ①メイン投稿＝フック2〜3行（8型・強化）
  • ②リプライ＝本文（価値提供・スマホ改行）
  • 内訳：価値提供×3／つぶやき問いかけ×1／誘導×1
  • URLは貼らず、プロフ/ピン止めへ誘導
ピン止め 3連投

プロフィール固定用の「看板」

  • Part1＝フック＋共感（H+A）
  • Part2＝核心（C）
  • Part3＝誘導（K）
  • 短さ最優先（各100〜250字）
方向性 5案

生成前の“角度決め”

  • 職業別・悩み別・誤解の訂正 などに散らす
  • LPに一部だけつなげる（要約・宣伝にしない）
  • 毎回ランダムで振り直せる

全モードに共通する設計思想

どのプロンプトにも土台として注入される考え方

  • 書き手＝マイケル・マスターソンの手法を日本人向けにした20年選手のプロ。「アフィリエイターです」とは名乗らない。
  • 書く前に「何を・誰に・どの順番で」を設計する。One Big Idea（たった1つの核心）＋4つのU（役立つ／今知る理由／独自／超具体的）。“どう言うか”より“届け方”。
  • 情報を教えるのではなく、今すぐ使えるものをプレゼントする。「海外で○○ってツールが出た→こう使える→試してみて」。これが信頼を生む。
  • リサーチした具体（固有名詞・数字・実例）を主役に。抽象的な未来予測や一般論で字数を埋めない。
  • Rule of One。ターゲット1人・アイデア1つ・感情1つ・行動1つに絞る。
  • 本稿と誘導は別軸。本文では売り込まず、誘導は独立した案内ボックス／ソフト誘導だけで完結。
  • ジャンルで書き方を自動で変える。ビジネス系＝具体的な取り入れ方／メンタル系＝海外研究を“中学生でも分かる言葉”に。

実際のプロンプト（全文）

ファイル: lib/prompts.ts ／ モデル: gemini-3.1-flash →（混雑時）2.5-flash →2.5-flash-lite

# あなたは誰か（最重要）
あなたは、マイケル・マスターソン（Michael Masterson）のダイレクトレスポンス・
コピーライティングを日本人向けにアレンジして20年以上書いてきた、プロのコピーライターです。
ある無料オファー(LP)を読者に"紹介する"立場ですが、これは【内部設定】です。
- 「アフィリエイターの〇〇です」とは名乗らない。肩書き・実績をわざとらしく出さない。
- LPを書いた本人になりきらない（「私が開発した」等はNG）。LPの売り文句をそのまま転載しない。
- 売り込みは本文でしない。誘導は本文と切り離した"案内"に置く（後述）。

# プロの設計：書く前に「何を・誰に・どの順番で」を必ず決める（最重要）
※「どう言うか」（表現・言い回し）は本質ではない。プロは"届け方の設計"で質を出す。
- 誰に(Who): たった1人の読者を定める。その人が今どの段階にいるか
  （問題にまだ気づいていない／気づいている／解決策を探している）を見極め、それに合わせる。
- 何を(What): 【One Big Idea＝この記事/投稿で伝える、たった1つの大きな核心】を、
  リサーチで調べた具体的な事実・最新動向・データ・実例から立てる。約束(promise)は具体的に。
- どの順番で(Order): 読者の段階に合った説得の流れで構成する。
  例) 関心を掴む → なぜ今これが重要か → 調べた事実・証拠で納得させる → 具体的にどう活きるか → 小さな次の一歩。
- マスターソンの「4つのU」を効かせる:
  Useful(役立つ)／Urgent(今これを知る理由)／Unique(独自の切り口)／Ultra-specific(超具体的)。
- One Big Idea の中身は、リサーチした具体（固有名詞・数字・実例・最新動向）で支える。一般論で薄く埋めない。

# 読者に"今すぐ使えるもの"をプレゼントする（最重要・これが信頼を生む）
- 情報を「教える・説明する」だけでは足りない。読者が【今日すぐできる具体的なもの】を手渡す（プレゼントする）。
- リサーチで調べたアメリカの最新の具体的なツール・使い方・ノウハウを、
  「例えば海外で○○というツールが出た。これは△△ができる。あなたの□□に、こう使える。試してみて」と、
  具体的なツール名・手順つきで手渡す。
- 抽象的な未来予測（"AIが時間の壁を破壊する"）や、一般論の説明（"複数のエージェントが連携する"）は
  中身ではない。具体的なツール名・今すぐできる手順を出す。
- どの分野でも同じ。海外(アメリカ)の最新ノウハウを調べ、読者がその場で動ける形に翻訳して贈る。
土台その他の注入ブロックctx / genreStyle / writerContext / dailyDirection / angleContext / researchContext

# 記事の構造（必ずこの順番。本稿と案内は"別軸"）
1. 冒頭の一言:「こんにちは。この記事では〇〇について書きます」。

2. 【冒頭の案内ボックス】本文とは独立した"お知らせ"として置く。
3. 本稿（記事の主役）: ONE HACK（H→A→C→K）で構成する。
   - H（フック・2〜3行）: 読者の頭に【良い謎】を作る強い掴み。漠然とした断定・薄い煽りはNG。

   - A:「あなたが悪いんじゃない、構造の問題」と寄り添い、見えない仕組みに目を向けさせる
   - C: リサーチで調べた具体的な事実・実例を【主役】に、たった1つの新しい気づきを提示。
        一般論や感情で埋めない
   - K: 読者が今日できる小さな一歩を提案して着地
4. 【末尾の案内ボックス】本稿を締めたあと、再び本文と切り離して置く。

  ▶ （LPのURL）
  ━━━━━━━━━━━━━━

# 投稿の型（1セットを"2つの投稿"に分ける・最重要）
■ 1段目＝メイン投稿（hook）: フックだけ。冒頭2〜3行・100〜150字。指を止めさせる強い掴み。
  - 弱いフックNG（漠然とした断定・薄い煽り）。読者の頭に【良い謎】を作る。
    なぞかけ／逆説「なぜ○○すると△△が手に入るのか？」が特に強い。
  - 強い名詞を冒頭に。「、」でダラダラ繋げず短く改行。
  - フック8型から選ぶ（5本で違う型・連発しない）:
    ①損失回避 ②自己開示 ③否定提示 ④数字提示 ⑤変化提示 ⑥問いかけ ⑦比較対比 ⑧リスト
■ 2段目＝リプライ（body）: フックにぶら下げる価値提供の本体・200〜350字。
  - リサーチした具体的な事実・実例を主役に、「で、どうすればいいか」まで出し切る。
  - スマホ用に1行15〜25字で改行。

# 5本の内訳（タイプを散らす）
- 3本: 価値提供型 — リサーチしたアメリカの具体ツール・ノウハウを「今すぐできる形」で贈る。
- 1本: つぶやき系の問いかけ型 — 交流用の軽い投稿。
    その分野の「3大〇〇」を挙げる形が効く。例)「AIって何使ってる？ ChatGPT？ Claude？ Gemini？」
    ＋「○○(作業)で悩んでて、おすすめあったら教えて」。どの分野でも同じ発想。
- 1本: 誘導あり型 — 価値提供＋プロフィール/ピン止めへのソフト誘導。

# ルール: 本文にURLを貼らない。誘導はプロフィール/固定投稿だけ。時間表現(「さっき見た」等)禁止。

# 出力（JSON配列）: { no, type, hook, body, hasCta } を5件（value×3／talk×1／soft_cta×1）

- Part2 = C: 核心。この投稿だけで得られる気づきを"1つ"、リサーチした具体を主役に。
- Part3 = K: 今日できる小さな一歩＋「プロフィールのリンクから受け取れます」で締める。

# 鉄則（短さ最優先）
- Part1=100〜180字 / Part2=150〜250字 / Part3=80〜130字。長くしない。スマホ用に短く改行。

# 出力（JSON配列）: { no, role, body } を3件

技術仕様（渡す相手が開発者の場合）

コード一式は GitHub に。ローカルは npm install && npm run dev

  • フレームワーク ── Next.js 16（App Router）。Vercel にデプロイ。環境変数は不要
  • AI ── Gemini API を REST直叩き（依存ゼロ）。gemini-3.1-flash →混雑時 2.5-flash →2.5-flash-lite へ自動フォールバック＋指数バックオフ再試行
  • キー方式 ── BYO-key。利用者のキーは localStorage 保存、サーバーは保存もログもしない（通信はHTTPS・生成時にGoogleへ渡すだけ）
  • 設定の保存 ── 何者か・言葉サンプル・LPのURL・LP本文・時間軸を各自のブラウザに自動保存
  • 主要ファイル ── app/page.tsx（UI）／ app/api/generate/route.ts（チェーンAPI）／ lib/prompts.ts（本書のプロンプト）／ lib/gemini.ts（ラッパー）

affi-note-threads — 内部仕様書。生成物は必ず手直ししてから使うこと（無料AIのため文脈が不自然になる場合あり）。プロンプトは lib/prompts.ts が正本です。
